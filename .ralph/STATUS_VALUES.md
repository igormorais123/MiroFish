# Status Values

- `todo`: captured but not ready.
- `ready`: clear enough for the executor.
- `in_progress`: claimed; must be resumed before new work.
- `blocked`: needs human/context/tooling.
- `manual-testing`: requires human or interactive verification.
- `done`: delivered and verified.
- `partial`: useful progress, not complete.
- `failed`: attempt failed and should be redesigned.
